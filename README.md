# Приложение для отслеживания ресурсов(как htop но в web)

Здравствуйте!
Этот проект предоставляет веб приложение для мониторинга системных ресурсов в реальном времени(как htop).
Работает на парсинге файлов `/proc`, соответственно, запустится только на OC с Linux.

**Backend** использует nlohmann/json.hpp, cpp-httplib. Скопировал их в проект для упрощения сборки.
Требует для сборки компилятор C++17, CMake 3.16, make 4.3(возможно подойдет более старый), posix thread library.

**Frontend** HTML CSS TypeScript. Ничего компилировать не нужно, ваш браузер исполнит main.js.

# Инструкции по сборке.
cd build

cmake ../backend/CMakeLists.txt

make

# Запуск сервера.
./server_monitor_system_metrix

# Использование.
- Открываете браузер
- вводите локальный интерфейс http://localhost:8085

# Комментарии.
Многопоточность явно не использовал. Метод collect MetrixCollector защитил мьютексом, из за того что httplib по умолчанию создает множество потоков и возникает data condition.
Распараллеливание сбора метрик, то есть вызова collect не нужно, так как для получения адекватных данных, между сбором системных тактов нужно подождать(в моем случае жду 1 секунду).
